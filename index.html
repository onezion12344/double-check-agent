Two Approaches

Route A vs Route B

There are two fundamentally different ways to solve AI hallucination. Only one works today.

Route A — Model Internalization

Train the model to know when to verify.
OpenAI o3/o4 · Claude · DeepSeek · Gemini

  • ❌ Expensive — retraining needed per model
  • ❌ Model-dependent — only works on latest models
  • ❌ Self-deception — model still judges itself
  • ❌ Years away from reliability
→

Route B — External Enforcement

Don't ask the model to verify. Enforce from outside.
Double-Check

  • ✅ Works with ANY model, TODAY
  • ✅ No retraining — plugin hooks
  • ✅ Independent verification — model can't cheat
  • ✅ MIT open source, production-ready

How it works

LLMs hallucinate. We catch it.

Every AI agent generates content at scale. Even a 5% error rate produces thousands of wrong facts daily. Double-Check sits between the model and the user, verifying every claim before it reaches anyone.

⚡

Pre-answer verification

Before the model responds, extract claims from the question. Time-sensitive items (prices, hours) force external search. Inline tagging: ✅ confirmed, 🔧 corrected, ⚡ disputed.

📋

Source attribution

Every claim traced to its origin. What came from source vs what the model guessed. The ⚠️ list is the verification target — not a flaw, a feature.

🔬

Cross-verification

Independent verification questions, not re-reading the model's output. Search until evidence converges. ≥2 independent sources per claim.

🎯

Impact analysis

Not all errors matter. 🔴 Critical (breaks workflow), 🟡 Experience (nice to fix), 🟢 Cosmetic. Only fix what actually impacts the user.

The Pipeline

5 phases. One responsibility each.

Information flows down, never back up. No phase searches and judges at the same time. No phase tries to do everything. Each round is auditable.

0
Dedup Gate
Emergency skip & duplicated content check
0.5
Pre-answer
Extract claims from question. Time-sensitive = force search.
1
SIFT Audit
Source attribution. ✅ from source · ⚠️ guessed · ❌ contradiction
2
CoVe + FIRE
Independent verification. ≥2 sources per claim.
3
FABLE Impact
🔴 Critical · 🟡 Experience · 🟢 Cosmetic
4
Truth Sandwich
What was said → What's wrong → What's correct → Source

Get started

Drop it in. It just works.

Double-Check runs as a native Hermes skill + plugin. Two modes: always-on (every turn) and deep audit (full 4-round). Also works with any LLM framework via system prompt.

⚡

Hermes Agent (1 minute)

cp -r skill/ ~/.hermes/skills/double-check/
cp -r plugin/ ~/.hermes/plugins/double-check/
hermes skill enable double-check
hermes plugin enable double-check

Plugin hooks into every turn. Skill triggers deep audit on dense responses (≥5 claims).

🔌

Any LLM framework

Copy the skill's system prompt into your agent configuration. Add the plugin functions as tool definitions. Works with Claude Code, Cursor, OpenClaw, Codex, and any OpenAI-compatible framework.
